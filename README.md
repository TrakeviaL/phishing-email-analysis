# phishing-email-analysis
Phishing email analysis project in progress
## Objective
Analyze a suspicious email to determine whether it contains phishing indicators and assess potential risk.
## Scenario
A user reports receiving an urgent email requesting password verification through an external link. The sender claims to be from the IT department.
## Analysis Process
1. Reviewed sender email address for spoofing indicators.
2. Inspected email header for domain inconsistencies.
3. Checked hyperlink destination for mismatched URLs.
4. Analyzed message tone for urgency and social engineering tactics.
5. Evaluated attachment presence and file type.
## Indicators Observed
- Sender domain did not match official company domain.
- Hyperlink redirected to a non-company external website.
- Message contained urgent language pressuring immediate action.
- Minor grammatical inconsistencies present.
## Risk Assessment
The email demonstrates multiple phishing indicators including domain spoofing and social engineering techniques.
## Conclusion
The email is classified as a phishing attempt.
Recommended actions:
- Do not click the link.
- Report the email to security team.
- Block sender domain.
- Conduct user awareness reinforcement.
## Skills Demonstrated
- Email header analysis
- Phishing indicator identification
- Social engineering detection
- Threat classification
- Incident documentation
